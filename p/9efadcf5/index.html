原理分析

启动解析

XMLConfigBuilder

  • 解析mybatis-config.xml

入口: XMLConfigBuilder#parseConfiguration

  • mappers节点，child有package和其他，package指定resource下的路径，其他类型支持resource、url、mapperClass三个属性之一
    • resource、url对应一个单独的mapper.xml文件，通过XMLMapperBuilder.parse()解析
  • parse之后得到Configuration对象
  • 通过Configuration构建SqlSessionFactory

XMLMapperBuilder

解析Mapper.xml文件的mapper标签内容

  • cache-ref: 复用其他namespace下的cache
  • cache: 构建二级缓存
  • namespace
  • parameterMap（已废弃）: 构建ParameterMapping，
  • resultMap: 构建ResultMap
  • sql
  • select|insert|update|delete：通过XMLStatementBuilder处理

XMLStatementBuilder

  • 处理include
  • 对于insert和update，处理selectKey，每个selectKey对应一个MappedStatement
  • XMLScriptBuilder#parseDynamicTags: 对每个语句进行切割
    • 纯文本部分为StaticTextSqlNode
    • 有${}或类似<where>标签的，为TextSqlNode或WhereSqlNode
    • 最后组合成MixedSqlNode，通过XMLScriptBuilder构造对应的DynamicSqlSource或RawSqlSource
  • 处理标签上其他属性
  • 优先使用SelectKeyGenerator
  • 最终得到MappedStatement对象
